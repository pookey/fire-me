## Project Overview

FinTrack is a personal net worth tracker. It tracks investment funds (savings, pensions, property) with periodic value snapshots and includes FIRE (Financial Independence, Retire Early) projection calculations.

## Architecture

Three independent packages (no monorepo tooling — each has its own `package.json` and `node_modules`):

- **`backend/`** — Lambda function handlers (TypeScript, bundled with esbuild). Four handlers: `funds`, `snapshots`, `fireConfig`, `import`. All share `utils/db.ts` (DynamoDB document client) and `utils/response.ts`. Route dispatch is done via `event.routeKey` string matching inside each handler.
- **`frontend/`** — React 19 SPA with Vite, Tailwind CSS v4, Recharts. Auth via AWS Amplify/Cognito. Pages: Dashboard, DataEntry, Charts, Fire, Funds, Import.
- **`infrastructure/`** — AWS CDK (TypeScript). Two stacks: `FintrackCertStack` (ACM cert in us-east-1 for CloudFront) and `FintrackStack` (eu-west-2: DynamoDB, Cognito, API Gateway HTTP API, Lambda, S3+CloudFront). Cross-region references enabled.

## DynamoDB Single-Table Design

Table name: `FinTrack`. Partition key: `pk`, sort key: `sk`. GSI1: `gsi1pk`/`gsi1sk`.

- Funds: `pk=FUND`, `sk=FUND#<id>`
- Snapshots: `pk=FUND#<id>`, `sk=SNAP#<date>`. GSI1: `gsi1pk=SNAPSHOTS`, `gsi1sk=<date>#FUND#<id>`
- Snapshots denormalize fund name and category at write time

## Common Commands

### Backend
```bash
cd backend && npm ci && npm run build    # esbuild bundle to dist/handlers/
```

### Frontend
```bash
cd frontend && npm ci
npm run dev       # local dev server (Vite)
npm run build     # production build (tsc + vite build)
npm run lint      # eslint
```

### Infrastructure
```bash
cd infrastructure && npm ci
npm run build     # tsc
npm run synth     # CDK synth
npm run diff      # CDK diff
npm run deploy    # CDK deploy
```

### Tests
```bash
cd frontend && npm test   # run vitest tests — MUST pass before deploying
```

### Full Deploy
**Important: always run `cd frontend && npm test` and confirm tests pass before running deploy.**
```bash
./deploy.sh       # builds backend, deploys CDK, builds frontend with CDK outputs, syncs to S3, invalidates CloudFront
```

### Data Import Script
```bash
cd scripts && npm ci && npm run import   # runs csv-import.ts via tsx
```

It is safe to delete all data from DynamoDB and re-import from the CSV at any time. Use `scripts/clear-table.sh` to wipe the table, then re-run the import. There is no user-entered data that isn't in the CSV — all snapshots originate from the spreadsheet.

## AWS Details

Configure your deployment in `.env` (see `.env.example` for the template):
- `AWS_PROFILE` — your AWS CLI profile name
- `FINTRACK_DOMAIN`, `FINTRACK_HOSTED_ZONE_ID`, `FINTRACK_ZONE_NAME` — your Route53 domain config
- Primary region: `eu-west-2` (cert in `us-east-1`)

After first deploy, `deploy.sh` auto-populates `API_BASE_URL` and `COGNITO_CLIENT_ID` in `.env`.

## Frontend Environment Variables

Set via `.env.production` (auto-generated by `deploy.sh` from CDK outputs):
- `VITE_API_URL` — API Gateway endpoint
- `VITE_USER_POOL_ID` — Cognito User Pool ID
- `VITE_USER_POOL_CLIENT_ID` — Cognito client ID
